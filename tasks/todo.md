### 2-2. iOS ウィジェット（手動誘導 + インセンティブ設計）

> **制約**: Apple は MDM でホーム画面・ロック画面ウィジェットの強制追加を禁止している。
> **方針**: インセンティブでユーザーに自発的に設定させる。

#### ステップ 1: VPN アプリ（iOS）開発

- [ ] App Store 審査通過を意識した VPN アプリ設計
- [ ] ウィジェット Extension（WidgetKit）実装
  - **ホーム画面ウィジェット**
    - ポイント残高表示（毎日確認動線）
    - 今日のクーポン表示（CPC/CPS 収益）
    - VPN 接続状態表示
    - 広告バナー枠（CPM 収益）
  - **ロック画面ウィジェット**（iOS 16 以上）
    - ポイント残高（小サイズ表示、時計上下に配置）
    - クーポン残数（数字 + アイコンのみ）
- [ ] ポイント・クーポン機能内蔵
  - ウィジェット設定完了でボーナスポイント付与
  - ポイント使用 → CPS 収益化の動線設計
- [ ] Push 通知連携（エル投げ経由）
  - MDM でアプリ配布後、エル投げが自動 LINE 送信
  - メッセージ内容:「ウィジェット設定で毎月ポイントがもらえます」
  - スクリーンショット付き 3 ステップ手順
  - 設定完了後のボーナスクーポン配布

#### ステップ 2: エル投げ（通知システム）との連携

- [ ] LINE Messaging API 連携
  - アプリインストール後、ウィジェット設定案内を自動送信
  - ウィジェット設定完了を検知してボーナス配布
- [ ] ウィジェット設定フロー案内（iOS 向け）
  ```
  ホーム画面を長押し
       ↓
  左上「＋」をタップ
       ↓
  アプリを選んでウィジェット追加（約30秒）
  ```
- [ ] ロック画面ウィジェット設定フロー案内（iOS 16+）
  ```
  ロック画面を長押し
       ↓
  「カスタマイズ」をタップ
       ↓
  ウィジェット枠をタップして追加（約20秒）
  ```
- [ ] 設定率 KPI 管理（目標: 50〜70%）

#### ステップ 3: 広告収益化

- [ ] ウィジェット広告枠を SSP に登録
  - ホーム画面ウィジェット: CPM バナー枠
  - ロック画面ウィジェット: ポイント残高（CPS 誘導）
- [ ] WidgetKit からの SSP API 呼び出し実装
  - Background refresh で広告フェッチ（TimelineProvider）
  - 表示インプレッション計測

## iOSウィジェット攻略方針（まとめ）

```
Step 1: MDM で VPN アプリを App Store 誘導（エル投げ経由）
         ↓
Step 2: VPN アプリにポイント・クーポン機能を内蔵
         ↓
Step 3: エル投げが「ウィジェット設定でボーナス」を自動通知
         ↓
Step 4: ユーザーが自発的に設定（インセンティブ効果）
         ↓
Step 5: ウィジェットが広告面になる
```

**Androidは強制、iOSはインセンティブ設計で同じ結果を目指す。**

## メモ・制約事項

- iOS WidgetKit: バックグラウンド更新頻度は OS が制御（1日数回）
- iOS ロック画面ウィジェット: iOS 16 以上のみ対象
- Apple ガイドライン: ウィジェット内に動画広告不可、静止画・テキストのみ
- SYSTEM_ALERT_WINDOW (Android): targetSdk 33 以上では権限ダイアログ必須

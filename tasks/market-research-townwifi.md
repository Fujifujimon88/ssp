## 3. 収益モデルの詳細

### 収益源構成

```
[収益Source 1] アプリ内広告収益（主力）
  ├─ 動画広告（リワード型）
  ├─ バナー広告
  └─ 収益化プラットフォーム: AppLovin「MAX」（複数アドネットワークの入札競争）

[収益Source 2] TownWiFi Ads（B2B広告配信サービス）
  ├─ Wi-Fi接続情報を活用した位置情報ターゲティング広告
  ├─ エリア指定（行政単位・市町村単位）、性別・年齢・時間・曜日指定
  ├─ PUSH通知配信（1通1円の明朗会計）
  └─ 来店計測・アンケート・効果測定機能付き

[収益Source 3] TownWiFi Analytics（B2B分析サービス）
  ├─ Wi-Fi接続データを活用したマーケティング分析
  └─ 集客効果測定・エリア分析レポート提供

[収益Source 4] ポイント交換エコシステム（間接収益）
  └─ ポイント発行でユーザー維持 → 広告視聴率・MAU向上 → 広告単価向上
```

### 収益化の成果数値
- DAU平均収益: 2021年8月〜2022年3月の8ヶ月で **8倍以上**に増加
- eCPM（有効インプレッション単価）: 2021年7月〜9月の2ヶ月で **60%以上**伸長
- 動画広告視聴率: 約**80%**（ポイント還元によりユーザーが積極的に視聴）

### ポイント還元システム
| 項目 | 内容 |
|------|------|
| ポイント名 | WiFiポイント（2021年7月20日リリース） |
| 獲得方法 | フリーWiFi接続、動画広告視聴、クイズ・ゲーム参加 |
| 交換先 | Amazonギフト券、nanaco、PayPayポイント、楽天ポイント、データ容量など |
| 累計発行実績 | 500億ポイント（約5億円相当、2023年8月時点） |
| 位置づけ | 広告収益を原資にしたユーザー還元（ユーザーは無料でメリットを受け取れる） |

### ユーザー層の変化とマーケティング
- 初期（〜2020年頃）: 20〜30代・都市部・ネットヘビーユーザー
- 現在（2022年〜）: 40〜50代・地方在住・節約志向層
- 効果的な広告訴求の変化:
  - 旧: 「通信費節約アプリ」
  - 現: 「街中にWiFiがこんなにあるのに損していませんか？（損失回避訴求）」
- 初期CPI（ダウンロード単価）: 30〜60円（資金調達後は120円程度に上昇）

## 4. データビジネスの詳細

### 収集データの全容（プライバシーポリシーより）

**必須収集データ**
- 端末情報: 端末機種名、端末メーカー、OSバージョン、言語・地域設定
- 利用状況: アプリ起動回数、位置情報許可の有無、通知閲覧日時
- 通信情報: キャリア、通信量
- 識別子: ユーザーID、広告識別子（IDFA / AAID）

**オプトインで収集するデータ（ユーザー同意ベース）**
- 端末位置情報（GPS）
- 性別・生年月日
- 通信契約内容・料金
- 周辺Wi-Fi情報一覧
- Wi-Fi接続・切断時刻、SSID、BSSID、スポット所在地、IPアドレス

**注意: IMEIの明示的な収集は確認されていない**

### 第三者提供先

| 提供先 | 提供情報 | 目的 |
|--------|---------|------|
| Wi-Fi提供者 | 性別・生年月日・任意追加情報 | Wi-Fi認証 |
| アプリ分析事業者 | 端末情報・利用状況データ | アプリ改善 |
| **マーケティング事業者**（株式会社フリークアウト含む） | Wi-Fi接続情報・位置情報・広告ID・性別・生年月日 | ターゲティング広告・マーケティング |
| ポイント交換先（イオンリテール、オプテージ、PayPay、サイバーエージェント） | ユーザーID・ポイント数 | ポイント交換処理 |

### データ活用の仕組み
1. ユーザーが街中のフリーWiFiに接続 → SSID・位置情報・時刻を記録
2. 「この人は今この店舗エリアにいる」と判定 → TownWiFi Adsで広告配信
3. 集積データをマーケティング事業者（フリークアウト等）に提供 → ターゲティング精度向上
4. 統計化・匿名化データをWi-Fi提供者やその他の事業者に提供

### データ価値の核心
- Wi-Fi接続ログ = 「実空間での行動履歴」（オンラインcookieではなく、リアル位置情報）
- 「今この場所にいる人」へのリアルタイム広告配信が可能
- 競合との差別化: GPSやアプリ起動の必要なく、Wi-Fi接続という「自然な行動」でデータ取得

## 5. VPN機能（WiFiプロテクト）の詳細

### 機能概要
- サービス名: **WiFiプロテクト**
- 機能: VPN接続によりフリーWiFi利用時の通信を暗号化
- 技術的仕組み: ユーザーのスマートフォン → VPNトンネル → インターネット（中間者攻撃・盗聴を防止）

### 廃止の詳細
| 項目 | 内容 |
|------|------|
| 廃止日 | 2022年6月30日 |
| 廃止理由 | WebのHTTPS化が広く普及し、VPNによる暗号化の必要性が相対的に低下 |
| 廃止後の代替 | SSL通信での対応。より高いセキュリティが必要なユーザーは別途VPNアプリを推奨 |

### ビジネス的な考察
- VPN機能は「安全なフリーWiFi接続」というユーザーメリットを提供するための付加価値
- VPN廃止後もコアアプリ（Wi-Fi自動接続）は継続 → 収益への影響は軽微と推測
- VPN維持コスト（サーバー費用・帯域コスト）削減も廃止の動機の一つと考えられる

## 6. ユーザーメリット設計

### アプリの価値提案
1. **自動接続**: 街中の200万スポット以上のWiFiに手動操作なしで接続
2. **品質フィルタ**: 速度が遅いWiFiや不安定なWiFiには接続しない
3. **ポイント還元**: 接続・広告視聴でポイント獲得 → 各種ポイント・ギフト券に交換
4. **通信費節約**: モバイルデータ通信量の消費を抑制

### App Storeレビュー評価
- 評価: **4.3/5.0**（169,835件、2025年時点）
- 肯定的な声: 「一瞬でWi-Fiに接続」「速度の遅いWiFiには繋がらない」「開発者の対応が良い」
- 否定的な声: 退会方法がわかりづらい、広告の読み込みが遅い、バグ・動画再生不具合

### ポイント還元インセンティブ設計の効果
- ポイント機能導入でアンインストール率が数%低下
- 動画広告視聴率が約80%（高エンゲージメント）
- 広告主にとって「視聴意欲のあるユーザー」への配信 → 高単価化

## 7. 類似サービス・国内外の事例

### 国内事例

**Boingo Wireless（米国）**
- 空港・軍施設・スタジアムの大型会場でWiFiネットワークを構築・運営
- Boingo Mediaプラットフォームで広告スポンサーシップ収益を獲得
- 年間10億以上のコンシューマーリーチ
- WiFi費用を広告収益で補助 → 会場側は無料Wi-Fi提供が可能に

**NTTドコモ「アドWi-Fi」（国内）**
- Wi-Fi接続時のWebページに自治体・企業広告を掲載
- 広告収入の一部をWiFi設置費用に充当 → 設置者のコスト削減

**OG Wi-Fi SPOT（国内）**
- 店舗向けフリーWiFi × ユーザーへの広告ページ誘導
- 飲食・美容・医療・ホテルなど向け

**WiFiモバイル型（Playwire等）**
- リワード動画広告をWiFi接続の代価として利用
- ユーザーは動画を見ることで無料接続時間を取得

### VPN × WiFi × 広告の組み合わせ事例

| サービス | 特徴 | 収益モデル |
|---------|------|-----------|
| Hotspot Shield（Pango/Aura） | 無料VPN × 広告表示 | フリーミアム（無料版は広告表示） |
| Opera VPN Pro | ブラウザ内蔵VPN × データ収益 | データ活用・有料プラン |
| Tenta Browser | VPN内蔵ブラウザ × 広告なし | プレミアムサブスク |
| タウンWiFi（過去） | フリーWiFi接続 × VPN保護（廃止）× 広告 | 広告収益 |

### MDMデバイス × VPN × 無料インターネット提供モデル
- 現時点で「MDM管理デバイス + VPN配布 + 無料WiFi提供」をパッケージ化した商用サービスの明確な事例は確認されていない
- タウンWiFiのアプローチが最も近い: アプリ（一種のプロファイル管理）でWiFi設定を一括制御 + VPN提供

## 8. MDMプラットフォームが模倣・参考にできる点

### A. 収益モデルの設計
1. **広告収益 → ポイント還元のループ構造**
   - MDM管理デバイスのユーザーに広告を表示 → 広告収益の一部をポイント・サービス割引として還元
   - 「使ってもらうほど収益が生まれる」モデル

2. **B2B広告配信プラットフォームの二次事業化**
   - デバイス管理データ（場所・業種・利用状況）を活用した広告配信サービス
   - 「どの業種のデバイスが何時に何を使っているか」という行動データは広告価値が高い

3. **ポイントエコシステムによるユーザーリテンション**
   - MDMエージェントのアクティブ使用・コンプライアンス遵守にポイント付与
   - ポイントをSaaS割引・Amazonギフト券等に交換 → 解約抑止

### B. データビジネスの構造
4. **Wi-Fi接続データ = 行動ログ**（模倣可能）
   - MDM配下デバイスの接続先SSID・位置情報・アプリ使用状況を蓄積
   - 統計・匿名化してマーケティングデータとして販売（要: プライバシーポリシーの整備）

5. **オプトイン型データ収集の設計**
   - 「便利な機能と引き換えにデータ提供に同意してもらう」設計
   - タウンWiFiは「自動接続」という圧倒的な利便性でオプトイン率を高めている

### C. VPN・セキュリティの収益化
6. **MDM × VPN配布の付加価値**
   - 管理デバイスにVPN設定をプロファイルで一括配布 → セキュリティ強化を「見える化」
   - VPN通過トラフィックで広告注入（技術的には可能だが倫理的課題あり）
   - より健全な方法: VPN接続先を自社サービスにして、ユーザーに付加価値サービスを提供

7. **「廃止」から学ぶ: 技術トレンドへの対応**
   - タウンWiFiはHTTPS普及でVPN廃止 → 「機能の旬が終わったら潔く廃止」の姿勢
   - MDMプラットフォームも市場変化に応じた機能の取捨選択が必要

### D. グロース戦略
8. **AppLovin MAX方式の広告収益最大化**
   - 単一アドネットワーク契約ではなく、複数ネットワークをウォーターフォール/入札競争させる
   - eCPMを最大化しながらフィルレートも維持

9. **ユーザー層の変化への適応**
   - タウンWiFiは「節約訴求」の変化でMAUを伸ばした
   - MDMでも「IT管理者向け」から「コスト削減を求める経営層」へ訴求層を広げることが有効

10. **30人で2500万DL規模の組織設計**
    - 小規模チームで大規模KPIを実現するため、インサイトドリブンな意思決定が鍵
    - A/Bテスト・ユーザー調査を意思決定の起点にする文化

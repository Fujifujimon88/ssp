## 1. Digital Turbine / Aura（旧Logia）

### ビジネスモデル
キャリア・OEM向けプリインストール型アプリ配信プラットフォーム。

- **On Device Solutions（ODS）**: デバイスセットアップ時の動的アプリプリロード（800万デバイス/月）
- **SingleTap**: ワンタップアプリインストール → コンバージョン率30%向上
- **App Growth Platform（AGP）**: アプリ開発者向けユーザー獲得ツール

**パートナー**: AT&T、T-Mobile、Samsung、Verizon

### 財務実績（FY2025）

| 項目 | FY2025 | FY2024 |
|-----|--------|--------|
| 売上高 | $490.5M | $544.5M |
| ODS売上 | $341.6M（70%） | - |
| GAAP純損失 | △$92.1M | - |
| 非GAAP調整済み純利益 | $36.8M | $60.3M |
| 非GAAP EBITDA | $72.3M | $92.4M |

**収益性**: ❌ GAAP赤字。売上減少傾向（-9.9% YoY）。**MDM型広告の米国先行事例として最重要参考事例**

### 黒字化が難しい理由
1. キャリア/OEMへのレベニューシェアが重い（ODS売上の大部分）
2. コンプライアンスコスト（FTC規制対応）
3. Google Playポリシー強化によるプリインストール制限

### 本プラットフォームへの示唆
- **⚠️ 警告**: Digital Turbineですら赤字。MDM型単独では黒字化困難
- **強み**: SingleTap（ワンタップCPI）の概念は価値あり → サイレントインストール×CVR最大化
- **差別化**: 店舗来店者という高LTVターゲットへの絞り込みが必須

## 2. Glance（InMobi傘下）

### ビジネスモデル
Androidロック画面を「スクリーン0」として活用する広告＋コマース統合プラットフォーム。

- **OEM深層統合**: Samsung、Xiaomi、Vivo、OPPO等にプリインストール
- **Shoppertainment**: AI駆動ファッション試着、インタラクティブコマース
- **SoftBank Japan連携**: 日本市場向けパートナーシップ（2023年〜）

### 財務実績（FY24）

| 項目 | FY24 | FY23 |
|-----|------|------|
| 売上高 | ₹614Cr（$73M） | ₹325Cr |
| 成長率 | +89% YoY | - |
| 純損失 | △₹929Cr | △₹1,094Cr（悪化傾向改善中） |
| 広告収入 | ₹336Cr（54.7%） | ₹248Cr |
| コマース収入 | ₹254Cr（39.6%） | - |

**収益性**: ❌ 赤字。ただし損失は縮小中。評価額$1.8B（Googleら出資）

### エンゲージメント指標
- DAU: 1億人以上
- 平均日次利用時間: **25分**（ロック画面でこの数字は驚異的）
- SoftBank Japan経由: 日本でのロック画面広告実験中

### 本プラットフォームへの示唆
- **コマース統合**: 広告（CPM）よりもトランザクション手数料（45-70%粗利）が高収益
- **ロック画面の価値**: 「スキップ不可＋高エンゲージメント」は本プラットフォームと同じ強み
- **赤字警告**: ロック画面広告単独では黒字化困難。**コマース統合が黒字化の鍵**

## 3. Xiaomi MIUI / HyperOS

### ビジネスモデル
ハードウェア5%低利益率からOS/サービス収益への転換。

- **GetApps**: ビルトインアプリストア（260M月次ユーザー）
- **広告**: OS統合広告（ロック画面、通知パネル、プリインストールアプリ内）
- **IoTエコシステム**: スマートホーム・ウェアラブルとの統合

### 財務実績（2024年）

| 項目 | 2024年Q3 | 成長率 |
|-----|----------|--------|
| インターネット事業売上 | RMB 8.5B | +15.7% YoY |
| **インターネット粗利率** | **76.6%** | （ハードウェア22.7%の3倍以上） |
| グローバルスマートフォン出荷 | 世界第3位 | - |

**収益性**: ✅ 黒字。インターネット事業粗利76.6%がビジネスモデルの核心

### 示唆となる粗利構造
```
ハードウェア（スマホ本体）: 粗利 ~5%
インターネットサービス（OS広告）: 粗利 76.6%
→ OS広告1件 ≒ スマホ本体15台分の利益
```

### 本プラットフォームへの示唆
- **粗利構造の差**: ハードウェア（デバイス提供）でなく、OS/ソフトウェア層で稼ぐモデルが正解
- **日本適用の壁**: OEM深層統合は困難。**MDMプロファイル経由のソフトウェア統合**が現実的代替
- **粗利目標**: インターネット事業型（70%+）を目指す = 広告配信の限界費用をほぼゼロに

## 4. OPPO ColorOS / Vivo / Realme（BBK Electronics傘下）

## 5. Samsung Knox + Samsung Ads

### ビジネスモデル
エンタープライズMDM（Knox）＋消費者向け広告データプラットフォーム（Ads）の二刀流。

- **Samsung Knox**: 企業向けMDMセキュリティ（本プラットフォームと類似の構造）
- **Samsung Ads**: ACR（自動コンテンツ認識）による50M+ Smart TV広告
- **データ統合**: TV視聴データ＋モバイル購買データの横断ターゲティング

### 財務実績（Samsung Electronics 2024年）

| 項目 | 2024年 |
|-----|--------|
| 総売上 | $211.2B |
| 営業利益 | $32.5B |
| 営業利益率 | 15.4% |
| Samsung Ads推定売上 | $335.7M |

**収益性**: ✅ 黒字。Samsung Adsは「第一者データの要塞」として競合優位

### ACRデータの強み
- 78M米国Smart TV（決定論的デバイスID）
- TV視聴→モバイル購買の連鎖計測
- Cookie廃止後の最強ファーストパーティデータ

### 本プラットフォームへの示唆
- **MDM ≒ Knox**: デバイス管理でデータ取得。同じ構造でAds展開可能
- **第一者データの価値**: 店舗POS×MDMデバイスデータ = Samsung ACRと同等の希少性
- **B2B×B2C二刀流**: 店舗向けMDMで基盤構築、消費者向け広告で収益

## 6. IronSource（Unity LevelPlay傘下）

### ビジネスモデル
アプリ向け広告ネットワーク仲介・マネタイズプラットフォーム。

- **LevelPlay**: 25+アドネットワークへの統一SDK、リアルタイムオークション
- **ILR（Impression Level Revenue）**: 各インプレッションの収益可視化
- **デバイスライフサイクル広告**: 開梱直後から使用全期間対応

**収益性**: ❌ Unity全社は赤字傾向（2024年に大規模リストラ）

### 本プラットフォームへの示唆
- **マルチネットワーク仲介**: 複数広告主への分散配信でeCPM最大化
- **透明性**: インプレッション単位の収益可視化 → 本プラットフォームの収益可視化機能に応用
- **OpenRTB対応**: すでに実装済みのOpenRTBが最重要資産

## 本プラットフォームへの戦略提言

### 黒字化のための必要条件

Digital Turbine・Glanceが示す通り、**MDM型広告単独では黒字化困難**。

黒字化できているモデルの共通点:

1. **データ独占**（Samsung: ACR / Xiaomi: OS統合データ）
   → 本プラットフォーム: 店舗来店者×POS購買データの一次データ

2. **コマース統合**（Glance: Shoppertainment 39%がコマース）
   → 本プラットフォーム: 広告クリック→店舗QR決済→購買完了のクローズドループ

3. **限界費用ゼロ構造**（Focus Media・Xiaomi: 設置後コストなし）
   → 本プラットフォーム: MDM登録後の広告配信コストはほぼゼロ

4. **AI最適化**（AppLovin AXON 2.0 / Glance AI個人化）
   → 本プラットフォーム: ML/TwoTower実装済み → ROAS証明→単価引き上げ

### 推奨フェーズ設計

```
Phase 1（現在）: MDM登録×ロック画面広告
  粗利目標: 50-60%（Focus Mediaモデル）
  KPI: CPM ¥800-2,000

Phase 2（6ヶ月後）: 店舗POSデータ統合
  粗利目標: 65-70%（Samsung Adsモデル）
  KPI: CPA ¥1,000-3,000（来店CVで証明）

Phase 3（1年後）: コマース統合（Glance Shoppertainmentモデル）
  粗利目標: 70%+（トランザクション手数料）
  KPI: GMV取り分 2-5%
```

### 日本市場特有の障壁と対策

| 障壁 | 原因 | 対策 |
|-----|------|------|
| OEM交渉困難 | 日本OEM市場縮小 | キャリア経由統合、MDMプロファイルで代替 |
| キャリア既得権 | 3社寡占 | 決済サービス統合で新規価値創出 |
| プライバシー規制 | APPI（個人情報保護法） | 店舗POS一次データ＋同意UI徹底 |
| 広告単価低下 | 日本市場飽和 | 店舗LTV・来店CVデータで高単価証明 |
